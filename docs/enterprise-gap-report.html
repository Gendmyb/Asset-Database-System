资产管理系统企业化适配与部署推进方案

📅 盘点日期：2026-07-19 🏷 当前版本：v0.2.0 👤 编制：项目经理

背景与目标

场景：约 1000 人中型企业，IT 资产价值超 2000 万元（预计设备数千台量级），拟将本系统作为日常设备管理平台，承载设备出入库登记、资产盘点、与域控（AD）同步人员名单等企业基础操作。

现状：系统 v0.2.0 核心业务闭环（台账/入库/领用/维修/盘点/折旧/报废/报表/Webhook/审计）已完成、终审 PASS，但形态偏向"单组织/小规模 IT 资产台账 + 审计 + 事件外发"。对企业级场景 12 项关键能力核实结果：8 项缺失、3 项部分、0 项完整满足。

结论：台账与盘点"骨架"齐全，可作为底座；企业化"血肉"（身份集成、移动、审批、提醒、通知）几乎全缺。直接上线无法支撑 1000 人场景。

三、建设 Backlog

Wave 1 — 上线门槛（P0，4–6 周）

  • G1 AD/LDAP 同步 + SSO（已确认有现成 AD 域）：新增 internal/auth/ldap/（go-ldap），定时同步用户/部门，登录走 LDAP bind，保留本地 admin 兜底，同步写入审计链。
  • G2 用户批量导入：复用 import_service.go dry-run 模式，新增 user_import_service.go + /admin/users/import。
  • G3 扫码 + 移动盘点：后端 go-qrcode 生成 + /assets/:tag/qrcode；前端盘点页加扫码输入与响应式。
  • G4 到期提醒：新增 internal/scheduler/（cron）扫 warranty_until/due_date 触发事件总线 → webhook 外发。
  • G5 Excel 导出：引入 excelize，export_service.go 增 ExportAssetsXLSX。

Wave 2 — 推广必备（P1，4–6 周）

  • G6 原生通知 internal/notify/（SMTP + 钉钉/企微/飞书）订阅事件总线。
  • G7 审批流：重建审批表与状态机，领用/报废/维修挂可配置审批节点。
  • G8 资产关系：资产表加 parent_asset_id，详情面板展示外设树。
  • G9 行级权限：扩展 OrgScope 为部门级 data scope。

Wave 3 — 成熟增强（P2，按需）

  • G10 合同/许可证模块、G11 PDF 报表、G12 PWA。

六、验证方式

  • AD 同步：手动触发同步任务，核对 users 表与 AD 一致；LDAP bind 登录成功。
  • 批量导入：1000 行测试 CSV dry-run → 正式导入 → 行数校验。
  • 扫码盘点：生成 QR → 移动端扫描 → 盘点计划状态正确流转。
  • 到期提醒：插入临近到期资产 → 触发 cron → webhook/邮件收到通知。
  • Excel 导出：导出后 Excel 打开，中文无乱码、字段完整。
  • 端到端：docker compose up -d --build → 登录 → 资产入库 → 领用 → 盘点 → 报表导出全链路跑通。

七、多 Agent 执行编排

每 Wave 固定 4 个 Agent（2 技术 + 1 审查 + 1 测试），由总项目经理指挥与门禁；技术 Agent 并行开发互不冲突的域，完成后串行跑审查 → 测试门禁；双绿后提交一次代码，commit 标注 Wave/缺口编号。
